#พล่ามไปเรื่อย เรื่องภาษาในเกม : วันนี้ตอนไปรวมกลุ่มคุยกับหลายๆคนในวงการบอร์ดเกมเนี่ย มีเรื่องหนึ่งเกี่ยวกับภาษา ว่า "ภาษาแบบเกม" มันอ่านเข้าใจยากสำหรับคนเล่นใหม่ (ทั้งไทย อังกฤษ) พอดีตอนนั้นยังคิดไม่ออกเลยไม่ได้พูดอะไรไปแต่ระหว่างกลับบ้านก็กลับมาคิดต่อ
 
 
คำว่าภาษาในเกมก็น่าจะเป็นแบบ if X is Y then Z ในภาษาไทยก็จะเขียนแบบ ถ้าหากมี "X" สามารถใช้ "Y" เพื่อ "Z" อะไรแบบนั้น ซึ่งสำหรับคนทั่วไปมันฟังไม่ลื่นหูแล้วจะรู้สึกสะดุดๆ ไปจนถึงคิดว่าแปลไม่ดี ก็มีเสนอแนวคิดว่าถ้าแปลหรือเรียบเรียงให้มันสละสลวยอ่านลื่นๆก็อาจจะช่วยในการเพิ่มอัตตราการเข้าถึงคนเล่นใหม่ๆ
 
 
อันนี้ในมุมส่วนตัวแล้วผมกลับคิดว่าภาษาแข็งๆแบบลอจิคนี้อ่ะดีที่สุดแล้ว เพราะมันไม่ก่อให้เกิดคำถามตามมา (ถ้าคุณคิดตามลำดับขั้น) แน่นอนว่าส่วนหนึ่งการที่ผมเป็นโปรแกรมเมอร์ด้วยล่ะ เวลาอ่านความสามารถที่มันเรียบเรียงความสำคัญแบบเงื่อนไขไว้ข้างหน้า แล้วตามด้วยผล มันทำให้ผมคิดว่าไม่ต้องต้องตีความเยอะนิหว่า อยากได้ธีมก็เอาไปลง favor text ก็ได้ หรือคือผมชอบ if X then Z มากกว่า You can do Z if you have X เพราะรู้สึกว่าก่อนจะรู้ว่ามันทำอะไรได้เราอยากรู้เงื่อนไขก่อน ถ้าของไม่ครบจะได้ไม่ต้องอ่านที่เหลือไรงี้ หรือถ้าเขียนให้มันสละสลวยพอเป็นภาษาไทยมันก็มักจะเปิดช่องให้พวก 'บรรเจิด' มาตีความต่อ
  
  
ในการอ่านการ์ดเกมส่วนมากผมเลยเป็นคนที่ไม่ค่อยงงความสามารถการ์ดเกมเท่าไร เพราะพวกนี้มันจะเขียนรูปแบบประโยคที่เป็น block มีเงื่อนไขชัดเจนแล้วก็กระฉับ เวลาคนงงส่วนมากคือไม่ยอมแยก block ข้อความแล้วมองจุดที่เป็นเงื่อนไขไรงี้ 
 
 
#ตรงนี้นอกเรื่อง หรืออย่างตอนอธิบายกฎเกมเวลามีคนถามอย่างเช่น "พี่ครับใน Brass ผมต่อรางจากตรงนี้ได้ไหม?" ซึ่งผมจะไม่ตอบว่าได้หรือไม่ได้ แต่จะถามว่า "แล้วมันเป็น Network ของคุณไหมล่ะ?" คือผมรู้สึกว่าถ้าเข้าใจเงื่อนไขแต่แรกคำถามแบบนี้ก็จะไม่เกิดอะไรทำนองนั้น ซึ่งคนที่ได้ฟังอาจจะรู้สึกว่ากำลังโดน 'กวนตีน' อยู่ก็ได้ว่าถามจะเอา Yes/No ทำไมมาถามกูกลับ แต่ในมุมผมเองก็รู้สึกว่าก็กำลังโดนกวนตีนอยู่เหมือนกันที่อธิบายกฎแล้วทำไมไม่ฟัง (ว่ะ)
